# Microsoft MarkItDown — DOCX + HTML + born-digital PDF -> Markdown.
# Legacy DOC is rendered to PDF by LibreOffice first, then passed to MarkItDown.
# Extras: docx (mammoth), pdf (pdfminer.six). HTML support is in the core package
# (markdownify). Audio / Azure / YouTube extras are intentionally excluded.
#
# 0.1.6 (over 0.1.5): PDF O(n) memory leak fixed (page.close()) — matters for batch;
# PDF tables -> aligned Markdown; partial-numbered-list fix; HTML deep-nesting
# RecursionError fix. No API change (helper uses .convert()/.text_content/.title).
# OCR-layer + Azure Content Understanding additions are cloud — intentionally unused.
# 0.1.6 also pulls in pdfplumber (MIT) as a dep — already in the image, so the
# planned layout-aware coordinate pass for PDF structure/titles needs no new dep.
markitdown[docx,pdf]==0.1.6
